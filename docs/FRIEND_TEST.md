# Umeboshi 友人テストガイド

## テストの目的

実際の医学生に Umeboshi を使ってもらい、README だけで迷わず使えるか検証する。

## テスト手順（友人に送る内容）

### 1. README を読んでもらう

https://github.com/Ark-Karim/umeboshi を開いて、README の手順に従ってセットアップしてもらう。
**絶対に口頭で補足説明をしないこと**（README だけで完結するかが検証ポイント）。

### 2. カードを作ってもらう

実際の講義資料（音声 or PDF）を1つ指定して、カードを生成してもらう。

### 3. Anki でレビューしてもらう

生成されたカードを実際に Anki でレビューしてもらい、以下の観点でフィードバックを集める。

## フィードバック収集項目

### セットアップ
- [ ] README だけでセットアップ完了できたか
- [ ] つまずいた箇所（具体的に）
- [ ] 所要時間

### カード品質
- [ ] 穴埋め（cloze）の粒度は適切か（細かすぎ/大雑把すぎ）
- [ ] 英語表記（furigana）は正確か
- [ ] 図表（visualAids）は役立つか
- [ ] 1講義あたりのカード枚数（多すぎ/少なすぎ/適切）

### 使用感
- [ ] カード生成にかかった時間（実測）
- [ ] Anki でのレビュー体験（one-by-one, タイマー, LLMチャットなど）
- [ ] 従来の手動作成と比べてどうか

### 総合
- [ ] 継続して使いたいか
- [ ] 改善してほしい点（自由記述）

## 定量データ（可能であれば）
