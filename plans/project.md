# Project Proposal (企画書)

## 概要

授業音声から Anki カードを自動生成し、専用の Anki notetype で学習するための
オープンソースツールキット。個人利用のツールとして開発し、実際に友人に共有した
ところ成績向上が確認された。これを匿名の OSS プロジェクトとして世界に公開する。

詳細な設計思想は `PRINCIPLES.md` を参照。本書はその思想を踏まえた
「何を・なぜ・誰のために作るか」の要約であり、`SPEC.md`(技術仕様)・
`GOAL.md`(実行規約)と対になる。

## 背景・動機

- 開発者自身が医学生であり、日々の講義を Anki カード化する負担を減らすために
  自作した。
- 友人に共有したところ、実際に成績向上が確認された。この一次的な実証が
  最大の強みである。
- 個人の利益や技術的な目立ちたさではなく、OSS を通じた世界への貢献を動機とする
  (`PRINCIPLES.md` 原則7)。

## 目的(優先順位順ではなく並列)

1. 世界中の医学生(将来的には学生一般)に無償で使ってもらう。普及の規模は
   期待しすぎない。
2. 技術コミュニティからのフィードバックを得る。
3. OSS の先駆者としての立ち位置を取る(Anki / AnKing のような先行事例に続く)。
4. 実績として提示できる形にする。
5. マネタイズの可能性はゼロではないが、主目的ではない。

## 想定ユーザー

- 第一次ターゲット: 医学生(自分と同じ課題を持つ層)。
- 将来的ターゲット: 学生全般(高校生を含む)。ただし普及のハードルは高いと
  認識している。
- 前提スキル: OSS の慣習として「エンジニアが開発し、誰でもインストールできる
  形で配布される」ことを目指すが、開発者自身はこの配布設計に強い自信はない。
  → だからこそ `SPEC.md` の Definition of Done(README 1ページで迷わず動く)
  が最重要の受け入れ基準になる。

## スコープ

含むもの:
- 音声 → 文字起こし → Anki カード生成の一連のパイプライン
- 専用 Anki notetype
- セットアップ手順(README 1ページ)
- 著作権フリーのサンプル教材・サンプルカード

含まないもの(当面):
- GUI / Web UI 化(将来検討、今回のロードマップの対象外)
- 医学的正確性の保証・ファクトチェック機構(`PRINCIPLES.md` 原則1により
  意図的にスコープ外)
- 積極的なプロモーション活動そのもの(告知チャネルの選定は別途検討中であり、
  本書・本ロードマップの対象は「公開できる状態にすること」まで)

## 差別化・独自性

技術的な新規性では勝負しない(`PRINCIPLES.md` により明言済み)。差別化点は:
- 医学教育ドメインに最適化された生成・notetype 設計
- 実証された学習効果(友人の成績向上)
- 「動くものを、誰でも迷わず使える形で出す」という実行力

## ライセンス・帰属

- コードは Apache License 2.0(AnKing のライセンスを継承)。
- notetype 等コンテンツも同ライセンスに準拠する方向で `SPEC.md` にて確定する。
- AnKing 由来の要素がある場合、NOTICE ファイルで適切に帰属表示する。

## 公開形態

- 匿名(無難なニックネームを使用)。個人が特定される情報(大学名、実名、
  録音した講義の出所など)は一切含めない。
