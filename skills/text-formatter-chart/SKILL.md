# フローチャート

## MUST

1-2 step で改行（タブレットの横幅を超えないように）
絶対量の増加、減少のみ、「↑インスリン」「↓GnRH」のように矢印で表現。状態変化は文字で書く（Bad: ↑ECL細胞、Good: ECL細胞活性化）
コードブロック（<code>）に入れる. <pre> は使用禁止。
<code> を2重で使ってもいい. 
<br> で改行する. <pre> が使用禁止であるため、必ず <br> を使って改行する. 

## SHOULD
フローチャートの1ステップの長さは、1文を超えない。体言止めで短くする。
フローチャートの矢印は右向き (──&gt;) **下向き矢印は使用不可**
`or, etc., e.g., !` などを使ってノートらしく簡潔に表現する.

# インライン挿入

## High-Yield の記述
- 独立した章としてはまとめず、該当する解説文章のインライン直下に配置する。
- 先頭に `&#x1F511;` を付記し、1〜2文の極めて簡潔な短文で記述する。
- *出力例*：
  `&#x1F511;COPⅡは小胞体からゴルジ体への順行性輸送を担う。COPⅠはゴルジ体から小胞体への逆行性輸送を担う。`

## Contraindications とその理由 の記述
- 独立した章としてはまとめず、該当する解説文章のインライン直下に配置する。該当する薬物や病態の解説直下にインラインで配置する。
- 先頭に `&#x1F6AB;` を必ず付記し、1〜2文の極めて簡潔な短文で記述する。
- *出力例*：
  ``&#x1F6AB;緑内障患者には禁忌である。`瞳孔散大筋刺激（散瞳） ──> 隅角狭窄 ──> 房水排出障害 ──> 眼圧急上昇`を引き起こすため ``

## Mnemonic の配置
- **表記形式**：Mnemonic は積極的に提示する。提示する場合は、該当するセクションの最末尾、または関連要素の直下に、独立したブロックとして配置する。
- 先頭に `&#x1F4A1;` を必ず付記し、1〜2文の極めて簡潔な短文で記述する。
- *出力例*: `&#x1F4A1;「しめじ」で覚える。`

# 表・フローチャートの選定と記法

## MUST

**対比軸の完全一致**：表内の各列・各行における評価項目（対比軸）は厳密に一致していなければならない。片方の項目にしかない評価基準をもう片方に混ぜることを禁止する。

**項目分割の禁止（メタデータヘッダー型）**：1つのセルの中に、改行を伴う複数の独立したトピックや推論を詰め込むことを禁止する。最上部ヘッダー行に「対比項目」を埋め込むなどして、縦軸・対比軸を極めてシンプルに保つ。セル内は列挙可能な短い事実や定量データ（単語・短句）のみに留め、説明的な推論は表の前後に文章として完全に切り離して配置する。

**列数（col）の上限制限**：表形式を使用する場合、列数は**最大3列まで**（3 col以内）とする。評価軸や属性が多岐にわたり4列以上になる場合は、表形式の使用を禁止し、階層化された箇条書き（Nested List）へ落とし込んで記述する。

**セル内読点の禁止**：表内の各セル（マス）の文末において、読点（。）を使用することを完全に禁止する。体言止め、または句点のない短句で完結させる。


**前置きの強制**：伝達経路のフローチャートを使用する場合、読み手の予測性を担保するため、そのフローチャートの文脈を必ず先に文脈文章で明記する。
- Example:

  ```md
  ## 金属依存性の細胞死
  特定の金属元素が 細胞内に過剰に蓄積することで引き起こされる 細胞死がある: 

  | 細胞死の名称 | 依存金属元素 | メカニズム・関連疾患 |
  |---|---|---|
  | **フェロトーシス (Ferroptosis)** | 鉄 (Fe) | 過剰な鉄が **脂質の過酸化** を引き起こして発生。『鉄のサビが広がるように細胞の壁がボロボロに酸化される現象』。ヘモクロマトーシス（鉄過剰症）や非アルコール性脂肪肝炎（NASH）、癌 の新たな治療標的として注目。 |
  | **キュプロトーシス (Cuproptosis)** | 銅 (Cu) | 銅が過剰に蓄積することで誘導。先天的に銅が蓄積するウィルソン病などの病態に関与。 |
  ```


## SHOULD

**フローチャートにするか表にするかの選定基準**
- 対比対象（疾患など）が3個以上存在、因果の矢印が2ステップ以内──>表を選択する
- 対比対象（疾患など）が3個以上存在、因果の矢印が3ステップ以上──>対比対象の数と因果の矢印のステップ数を比較し、対比対象が多ければ表、因果の矢印が多ければフローチャートを選択する
- 対比対象（疾患など）が2個以内、因果の矢印が2ステップ以上──>フローチャートを選択する
- 対比対象（疾患など）が2個以内、因果の矢印が1ステップ──>表を選択する
- **フローチャート（矢印）重視の基準**：用量変化に伴う作動受容体や効果の切り替えにおいて、因果の矢印が3ステップ以上挟まれている（複雑な分岐動態を伴う）場合は、対比よりも「条件分岐ネストフロー」を重視して記述する。
  * Example: 
    ```html
    <code>
     ドパミン（医薬品）投与<br>
     ├─ [低用量] ──> 腎・腸間膜のD1受容体（タンパク質）刺激 ──> 血管拡張（腎血流保護）<br>
     └─ [高用量] ──> α1・β1受容体（タンパク質）刺激 ──> 血管収縮・心機能亢進（血圧上昇）
     </code>
    ```
- **比較（表）重視の基準**：変化の段階（低用量、中用量、高用量など）の対比行が3個以上存在する場合は、網羅的な比較を重視して「表形式」で整理する。

**Clinical Correlation（臨床的意義）のフローチャート化**
- 表記形式：病態メカニズムから臨床疾患・症状への移行は、作用機序のフローチャート（`──>`）として表現する。
- *出力例*：
  ```html
  <code>
  交感神経亢進 ──> β1受容体過剰刺激<br>
  ──> 心筋酸素消費量↑ ──> 心不全悪化（予後不良）
  </code>
  ```


**二項対立 ──> 原則、表にする**：フォワード遺伝学 vs リバース遺伝学のように、定義や探索の方向性が明確に対比される二つの概念は、表形式を prefer する。





## COULD

**物質が往復またはループする動態は、必ず循環型ステップフロー（`A ──> B ──> A`）の形式で完結させる。**

**フローチャートの階層化**：必要に応じて、フローチャート自体をインデントによって階層化（ネスト）して表記してもよい。

**方向ラベルの付記**：フローの文末に、その動態の機能的意味を示すラベル（``逆行`、`回収`、`リサイクリング` など）を `（） ` で適宜併記する。
- e.g. : `LDL受容体が粒子を取り込む [細胞膜] ──> 酸性環境で粒子を解離 [エンドソーム] ──> 受容体が再輸送され次の粒子を待つ (リサイクリング) [細胞膜]`


**相互作用（併用効果・作用反転）**: 併用する各薬物の作用機序（アクセルとブレーキ等）をフローチャート内で並列の箇条書きで示し、最後に結論（∴）を導き出す。
- example: 
  ```md
  **β2作動薬（医薬品）**: アデニル酸シクラーゼ（酵素）を活性化し、cAMP（セカンドメッセンジャー）の「産生」を増やす。
  **テオフィリン（医薬品）**: ホスホジエステラーゼ（酵素）を阻害し、cAMP（セカンドメッセンジャー）の「分解」を防ぐ。
  ∴ 両者の併用により細胞内cAMP（セカンドメッセンジャー）濃度が相乗的に高まり、気管支が強力に拡張する。
  ```

# 箇条書き

## MUST

**箇条書き（List）における主述の完全性**
- 情報密度を極限まで高める短文箇条書きであっても、各文には必ず主語と述語（動作主と作用対象、および結果）を含める。因果関係や論理の主体が消失するような、単語やフレーズのみによる「文の断片化」を完全に禁止する。
- Example:
  ```md
  ## オートファジー細胞死
  **オートファジー (Autophagy)** は、通常は細胞を「生かす」ための仕組みである。
  1. 栄養飢餓への対応: 自分自身のタンパク質を分解し、アミノ酸（エネルギー源）を作り出す。『飢えを凌ぐための非常食』。生後直後の 母乳を飲むまでの間、生存に必須の働きをする。
  2. 品質管理: 古くなった ミトコンドリアや 細胞内に侵入した細菌を分解・除去する。この特異的な分解を ミトファジーと呼ぶ。
  ```

## SHOULD

**高次構造やコンポーネントの階層表現**
- 一次〜四次構造や、シャペロン複合体の構成因子（GroEL/GroESなど）のような階層的・複合的なコンポーネント要素を記述する際は、**階層化された箇条書き（Nested List）** を使用する。
