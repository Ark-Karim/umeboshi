# 文体ルール
## 文体・トーン
- 常体（「である」「だ」「〜ない」）を徹底する。
- 受動態を避け、「〜が〜を行う」「〜を廃棄する」のように、誰が何を対象に行動するのかを明確にした能動的な表現を用いる。
- 行動を促す際、「〜しなさい」「〜せよ」という命令形ではなく、「〜するのが好ましい」「〜を推奨する」といった、合理的な選択肢としての提示に留める。
- 「これ」「それ」を使用する場合は、直前の1行にある具体的な名詞のみを指すようにする。文脈全体のニュアンスを指示代名詞に含めない。
- 箇条書きの項目や、事実のみを列挙するセクションでは、文末を「動詞＋する」ではなく体言止め（名詞・代名詞結び）にする。全体の文末の3割以上を体言止めにすることで、記述を引き締める。
- 一文（文頭から「。」まで）の長さを、原則として40文字〜60文字程度に収めること。80文字を超える長文は分割し、情報密度を保ったまま短読できるようにする。

## 文内容
- 原則に対する例外や、万が一のケースを記述する際は、「〜でない場合は」という表現で始め、その際の代替行動を直後にセットで記述する。

## 簡潔さ
   - 前置き、つなぎ言葉、不要な遷移表現は一切省き、直ちに答えや行動から記述を開始する。
   - 1文を極めて短くし、直接的な表現を選択する。冗長な説明は排除し、理解に不可欠な内容のみに留める。

## 客観性と事実の重視
- 感情的な肯定や最上級表現は一切禁止。
- 感情、感想、主観を排除し、事実、手順、定量データ、論理的因果関係のみを記述する。

## 線形な文構造
   - 読み手が読み返さずに一発で理解できるよう、意味の逆走を避けた、直截的でシンプルな文構造を徹底すること。

# ドキュメント構造化ルール
## Markdownによる厳格な階層化
- 内容はすべて `#`, `##`, `###` の見出しと、箇条書き（`-`）を用いて階層化・構造化すること。 dense な長文の壁は作らない。

## セクションの書き方
- 単語の羅列ではなく、意味のある文脈ごとにセクションを区切る。
- 1つの段落は原則1〜3行に収める。
- 本筋から外れる例外処理、過去の経緯、実用上のコツ、注意点は、本文中には混ぜず、項目の最下部に「※ 」から始まる注記として独立させる。

## 条件分岐の形式化
- 「〜の場合」「〜の時は」といった条件による分岐は、箇条書きと「対象: 」「午前: 」などのコロン区切りを用いて構造的に明示。
- メリット・デメリット、手順（午前/昼/午後など）を比較・整理する場合は、箇条書きで並列な項目として明示する。

# 思考の参考とする執筆例文
```md
以下、面の補修について記述する。新たな間仕切り作成は63代(2020年)が最後であり、私は設計上の細かい数字などは知らない。それについては63回生の里村さんが詳しいだろう。

### 劣化した間仕切りの判別
ラワンの劣化度は色では判断できない。そこで、以下のように区別する。

- 使用可能なもの: 板をたたいてみたときにしっかりしていると感じるもの
- 劣化しているもの: 板を叩いてみて簡単にへこむもの

※ 間仕切りは上からロール紙を巻いて用いる。したがって、そこまできれいな状態に保っていなくても実用上は問題ないだろう。
```

```md
手順
- 午前: 文実管理の備品移動
- 昼: 終礼
- 午後: 部所有備品の備品移動

### プラスチック段ボールについて

メリット
- はるかに軽い: 非常に軽く、間仕切りを運ぶのが楽になる。
デメリット
- 紫外線で劣化する: 直射日光に当てると2年くらいでボロボロになって使えなくなる。

※ あまりに分別が面倒臭いゴミは、部室内にためておき、駒場東大前のWifi付きゴミ箱に捨てに行く手もある。
```
