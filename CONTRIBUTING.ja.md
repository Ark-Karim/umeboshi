# Umeboshi へのコントリビューション

> [English version](CONTRIBUTING.md) | 日本語版

Umeboshi は Claude Code のスキルシステム上に構築された AI 駆動の Anki カード生成ツールです。
誤字の修正から、新しい文字起こしバックエンドの追加、カード生成品質の改善まで、コントリビューションを歓迎します。

プロジェクトの設計思想の概要については [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md) を参照してください。
すべての判断の指針となる原則については [plans/PRINCIPLES.md](plans/PRINCIPLES.md)
（開発者の母語である日本語で記述）を参照してください。

## 募集しているコントリビューション

何から手を付けていいかわからない場合は:

- **新しいバックエンド**: 新しい LLM プロバイダー、文字起こしサービス、Anki インポート方法のサポートを追加してください。
  ステップバイステップのガイドは [docs/PLUGIN_DEVELOPMENT.md](docs/PLUGIN_DEVELOPMENT.md) を参照してください。
- **バグ報告**: 不具合を見つけましたか？ [Issue を作成してください](https://github.com/Ark-Karim/umeboshi/issues)。
- **ドキュメント**: 誤字、不明瞭な箇所、未翻訳部分（特に英語→日本語）の改善。

## 大規模な変更について

大規模な変更に着手する前に、まず [Issue を作成](https://github.com/Ark-Karim/umeboshi/issues)
して議論してください。プロジェクトの方向性と合わない変更に労力を無駄にするのを防ぐためです。

## まずプラグインを検討する

コアパイプライン（`skills/` 内のメインスキル）を変更する PR を提出する前に、
そのアイデアが**プラグイン**として実装できないか検討してください。

Umeboshi は 5種類のバックエンドを持つプラグインシステムを採用しています:
- **transcribe** — 音声 → テキスト
- **generate** — テキスト → Anki カード
- **import** — カード → Anki
- **notetype** — カードの描画
- **sync** — クラウド同期（オプション）

プラグインは `plugins/<type>/<name>/SKILL.md` に置かれ、`plugins/registry.json` を通じて自動検出されます。
プラグインはたった1つの Markdown ファイルです — ビルドステップもフレームワークも不要です。
詳細は [docs/PLUGIN_DEVELOPMENT.md](docs/PLUGIN_DEVELOPMENT.md) を参照してください。

**目安**: 新しいサービス、プロバイダー、フォーマットを追加する変更はプラグインにすべきです。
既存のバックエンドの動作を変える変更はコアの変更です。

## Pull Request の説明

Pull Request を作成する際は、以下を含めてください:

- **何を**変更するのか
- **なぜ**必要なのか（該当する Issue があればリンク）
- **どのようにテスト**すればよいか（ステップバイステップの手順）
- 見た目の変更（カード描画、ドキュメントなど）については**スクリーンショット**や出力例

## Issue との紐付け

バグ修正や機能追加の PR は、既存の Issue を参照すべきです。
Issue が存在しない場合は、コーディングに時間を費やす前にまず Issue を作成し、
問題や機能の方向性について確認を取ってください。

## コードスタイル

### スキル (SKILL.md ファイル)

- メインの指示セクションとして `## CoT`（Chain of Thought）を使用する
- 入出力ファイルのパスを明示する — `<absolute path>` プレースホルダーを使用する
- 他のスキルを参照する際はフルパスを使用する: `skills/<name>/SKILL.md`
- スキルは短く焦点を絞ること。1つのスキルは1つのことをうまくやるべきです。

### プラグイン

- `plugins/INTERFACE.md` のインターフェース規約に従う
- 前提条件（APIキー、インストールが必要なツール）を `Prerequisites` セクションに記載する
- 外部APIを使用する場合はコスト見積もりを含める
- `plugins/registry.json` で `active` を変更し、パイプラインを実行して出力を確認することでテストする

## 1つずつ

各 PR は1つの Issue または機能に対応させてください。無関係な変更は別々の PR に分割してください。
小さく焦点を絞った PR はレビューが容易で、マージも安全です。

## AI 支援によるコントリビューション

Umeboshi 自体が Claude Code で構築されており、AI 支援によるコントリビューションを歓迎します。
ただし、**提出するすべての変更を理解している**必要があります。
自分の変更が何をするのか、パイプラインとどのように連携するのかを説明できない場合、PR はクローズされます。

AI 支援 PR のガイドライン:
- 提出前に AI 生成コードをすべて注意深くレビューする
- 参照されているファイルパスが実際に存在することを確認する
- 変更をエンドツーエンドでテストする（clone → install → generate → import）
- AI 生成 PR には `ai-assisted` ラベルを付ける

人間によるレビューなしで提出されたと思われる PR — 存在しないファイルへの参照、
実行不可能な手順、ドキュメント化されたインターフェースと一致しないコードを含むもの — は、
さらなる議論なしにクローズされる場合があります。

## テスト

提出前に以下を確認してください:

1. **スキルが正しくコピーされること**: `./install.sh` を実行し、すべてのスキルが `~/.claude/skills/` に表示されることを確認する
2. **プラグイン検出が機能すること**: `plugins/registry.json` で対象プラグインタイプの `active` を変更し、パイプラインを実行して新しいプラグインが呼び出されることを確認する
3. **シークレットスキャンが通ること**: CI ワークフロー（`gitleaks`）が PR でパスすること
